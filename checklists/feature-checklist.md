# Feature Checklist

> Task fields, acceptance-criteria format, and the canonical Definition of Done
> are defined in `docs/task-schema.md`. This checklist is a quick per-feature
> aid; the "Definition of Done" section of `docs/task-schema.md` is
> authoritative for completion.

## Requirements

- Business value identified
- Acceptance criteria defined
- Dependencies identified

## Architecture

- Architecture reviewed
- Technical decisions documented

## Development

- Code implemented
- Existing patterns followed
- No unnecessary complexity

## Testing

- Build passes
- Tests pass
- Edge cases reviewed
- Regression testing completed

## Documentation

- CLAUDE.md updated if needed
- architecture.md updated if needed
- decisions.md updated if needed

## Delivery

- Task marked complete
- Commit created
